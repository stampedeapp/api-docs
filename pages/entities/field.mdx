# Field

A form field in the booking, most often containing answers to questions asked during the booking form. A field contains detailed information about the data provided by the booker.

```json
{
  "id": "dateOfBirth",
  "name": "Date of Birth",
  "value": "1995-12-31"
}
```

### Properties

<Property name="id" type="string">
  The unique ID for each booking field.
</Property>

<Property name="name" type="string">
  The name of the booking field.
</Property>

<Property name="value" type="string">
  The value provided by the booker for this booking field.
</Property>

## Property Types

In theory any property type can be added to a booking. However, here are common property types that are used in bookings:

| Property ID    | Description                                                          |
| -------------- | -------------------------------------------------------------------- |
| `firstName`    | First name of the participant/applicant.                             |
| `lastName`     | Last name of the participant/applicant.                              |
| `emailAddress` | Email address of the participant/applicant.                          |
| `dateOfBirth`  | Date of birth of the participant/applicant.                          |
| `status`       | Status of the booking, see [BookingStatus](/entities/bookingStatus). |
